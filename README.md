## 幻觉分类定义
## 1. 幻觉分类体系
| 类别 | 说明 | 严重等级 |
|------|------|----------|
| **政策与规则编造** | 虚构或篡改退货、发票、优惠、发货等官方政策/规则。 | 中 |
| **产品参数编造** | 虚构或更改产品规格（蓝牙版本、材质、接口、功能等）。 | 中 |
| **能力越界** | 声称具备系统实际不存在的功能（物流查询、订单修改、退款进度查询等）。 | 高 |
| **安全与合规误导** | 提供可能危害用户健康或违反法规的建议（如孕妇可用含慎用成分的产品）。 | 高 |
| **信息遗漏与错误** | 遗漏关键信息（如尺码偏差提示）或给出错误信息（如关联品牌）。 | 低 |

## 检测方法
采用 LLM + 规则后处理 的自动化检测方案，流程如下：

构造 Prompt：将用户问题、系统回复、知识库拼接，要求模型判断是否出现幻觉以及具体类型（若存在）。

调用 LLM API：使用 qwen3.5-flash（温度 0.1，追求确定性输出），返回 JSON 格式结果。

规则校验：针对已知“能力越界”场景（如知识库为“无”却给出具体信息），进行二次规则校验以提高准确率。

输出：每条回复得到 is_hallucination 和 hallucination_type
